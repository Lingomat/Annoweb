16/4/2016 — 0.920

The data wipe/reset buttons in the 'settings' page hidden behind the debug. A toast tells the user when debug is on/off when the click on the beta version logo. Improved message when joining a track when the user has data (doesn't say deleting the annotation). Annotation UI starts in first region automatically.

The audio playback preference works correctly. A toast appears if the user tries to play audio after turning off both the source and secondary. Playing before the end of the last region would erroneously begin marking a region. We now allow the press and hold mode, but do not create a region.

Fixed issue where user could delete regions (with escape) for annotations that had joined a respeaking/translation track. This would orphan audio recordings. At the moment this triggers a toast with an overly long message. Regions beyond the availability of secondary audio may be deleted as normal. Fixed invalid seek to region on switching tracks where that track cursor was not in a valid region.

Error reporting for audio files. Aikuma will attempt to decode the audio through Web Audio API. If this fails the user is informed. Playback button enabled when importing a file. Export of WebVTT and SRT files is now done through Chrome file API when running as a Chrome Packaged App. This was necessary to open a file selector rather than saving in some unknown location.

New data structure upgrade system. This is necessary for any changes to underlying data. The first use case is a move to record an string for the language selector in addition to the ISO code. How people choose to refer to their languages is not something we should enforce based on ISO codes. Language selector upgrade to new system.

First steps to address performance issues. Underlying dependency versions were rolled up, such as Angular and Angular Material. Angular Translate filters have been moved to directives in key locations (more to do). Google Closure Compiler again enabled in whitespace mode, simple optimisations will need to wait until recorder.js is modified or replaced. Build uses CSS Clean. Culled alpha changes.
